# Smart objects (Photoshop placed layers)

Deep reference for smart-object work: model, PSD I/O, edit/replace contents, linked files, authoring, and the Photoshop 2026 ground-truth captures. Warp-specific machinery (warped SoLds, warp styles, the Warp Transform tool, Warp Text) lives in docs/warp.md. The never-save-over-PSBtest warning and the context-menu standing rule also appear in AGENTS.md.

A smart object stays a **`LayerKind::Pixel` layer** (the text-layer pattern) whose pixels are Photoshop's rendered preview; identification is the `layer_is_smart_object()` predicate over `patchy.smart_object.*` metadata (core/smart_object.hpp owns the keys and helpers, like adjustment_layer.hpp does).

- **Model**: per-layer placement metadata (source uuid, Trnf quad, size, dpi, lock reason, raster_status, block_dirty) parsed from the 'SoLd' (authoritative; 'SoLE' alias) or legacy 'PlLd' blocks by `src/psd/psd_smart_objects.{hpp,cpp}`; the source payloads live in `DocumentMetadata::smart_objects` (a `SmartObjectStore` parsed from the global 'lnk*' blocks, one `SmartObjectSource` per element with **shared_ptr-held bytes so the 40-deep whole-Document undo snapshots share one copy** — before this, every snapshot deep-copied embedded files). Unparseable blocks/elements stay opaque and re-emit verbatim.
- **Native Smart Filter model**: `Layer::smart_filter_stack()` is the per-instance semantic model of SoLd `filterFX`; `src/core/smart_filter.hpp` models stack/entry enable state, native order, blend mode/opacity/colors, Gaussian Blur, High Pass, Median, Dust & Scratches, and Surface Blur settings, and the one shared filter mask separately from the ordinary layer mask. `DocumentMetadata::smart_filter_effects` owns Photoshop's document-global `FEid`/`FXid` render caches, associated by the SoLd `placed` uuid rather than the shared source `Idnt`. Raw record bodies and decoded mask samples are shared across whole-document undo snapshots. A wholly supported Gaussian Blur/High Pass/Median/Dust & Scratches/Surface Blur stack with one unambiguous semantic FEid record is editable; an unsupported entry or cache keeps the whole layer on Photoshop's preserved preview.
- **Editable native rendering**: `src/filters/smart_filter_renderer.{hpp,cpp}` is the deterministic CPU path for supported native Smart Filters. Gaussian Blur embeds the current stack result in the transparent document-space FEid cache canvas, filters premultiplied RGBA, repeats the canvas edge like Photoshop, and alpha-trims its expanded result. High Pass operates on the exact current result bounds, repeats that result's nearest RGB edge, preserves alpha and bounds exactly, and computes `clamp(source - byte-rounded Gaussian + 128)` per channel, including hidden RGB under transparent pixels. Median also keeps the exact bounds. It independently takes the upper median of each RGBA channel over a square, edge-clamped window whose integer radius is the floor of the stored value. Before filtering RGB, transparent source pixels borrow straight color from the nearest nonzero-alpha source using squared Euclidean distance with down-then-right tie breaks; all-transparent inputs retain their hidden RGB. This keeps transparent edges from pulling black into the result. Dust & Scratches reuses that square RGB median and extension, keeps alpha and bounds byte-identical, and replaces the whole RGB triplet only when its maximum channel difference from the extended source is strictly greater than the integer threshold. Surface Blur pads by its rounded effective radius, applies an edge-clamped square weighted filter independently to straight RGB and alpha, then alpha-trims the result. RGB uses the same nearest-visible extension; each sample `v` around center `c` has weight `max(0, 5 * threshold - 2 * abs(v - c))`, and the quotient rounds to nearest with ties to even. Gaussian followed by any bounds-preserving filter uses the Gaussian result's trimmed bounds, while a bounds-preserving filter followed by Gaussian can still expand across the FEid canvas. The renderer composites each entry over the previous stack result using its opacity and blend mode, then applies the document-space shared mask once to the completed stack. Normal at 100% uses Photoshop's replacement path. The renderer is called only while producing a cached layer raster, never from `paintEvent` or the compositor. Setting changes, enable toggles, transforms, warp changes, Edit Contents, linked-source updates, Relink, and Replace Contents all regenerate from the unfiltered source instead of filtering the previous preview.
- **Smart Filter preservation**: untouched SoLd and FEid/FXid payloads re-emit byte-for-byte. Parsed version-1 cache records expose 8-bit raw or PackBits filter masks while retaining their raw bodies; an unknown entry, malformed cache, unsupported mask compression/depth, missing association, or duplicate `placed` association marks the whole stack unsupported. One opaque FEid/FXid block makes associations and mutations fail closed across the whole store because a hidden duplicate cannot be ruled out. PackBits row lengths are strict, and an invalid cache layout stops before optional-mask decompression. Never execute a supported prefix of an unsupported imported stack. Photoshop includes zero alignment bytes inside the declared FEid/FXid payload length; the codec accepts only the verified 0-3 zero-byte tail and reproduces that internal four-byte alignment after a rekey. Patchy-authored filters write a canonical Photoshop-shaped `filterFX` descriptor and a version-3 FEid block containing a full-document, PackBits-compressed unfiltered RGB/alpha cache plus the shared mask. Same-size edits patch known leaves in place. Mask painting replaces only the optional FEid mask tail, preserving the unfiltered cache prefix, record order, block dialect, declared-length form, and alignment; an unsafe or ambiguous record rejects the edit. Per-filter opacity and blend-mode edits patch SoLd only and leave FEid/FXid bytes unchanged. Append, duplicate, reorder, and delete rebuild only `filterFXList`, deep-cloning retained native objects so unknown fields, key order, and ID forms survive without shared descriptor aliases. No private `plSF` fallback exists.
- **Preserve-unless-edited**: untouched layers re-emit their original SoLd/PlLd blobs byte-identically; a move/transform sets `block_dirty` and the writer REGENERATES the blocks patch-in-place via the generic descriptor writer (psd_descriptor.cpp `write_descriptor`, which preserves key order and each id's charID/stringID form — Photoshop writes the 4-char key "warp" as a stringID, so id form is NOT derivable from length; `DescriptorObject::key_order`/`KeyEntry::long_form` carry it). `psd_descriptor_writer_round_trips_sold` pins read->write byte-identity.
- **Lock classification** (kLayerMetadataSmartObjectLock): unsupported warp/quiltWarp, non-affine quads, unsupported or ambiguous filterFX/FEid data, external ('liFE') sources, PlLd-only ("legacy"), and parse failures are preview-locked: Patchy shows/preserves Photoshop's raster and refuses pixel edits; rasterize always works. Photoshop Blur Gallery's `blurbTransform`, including native Tilt-Shift, remains in this preservation-only class. A fully supported Gaussian Blur/High Pass/Median/Dust & Scratches/Surface Blur filterFX/FEid pair clears the provisional filter lock. Empty lock = editable. (Supported warps unlock - see docs/warp.md.)
- **UI semantics**: a smart-object badge button on the layer row's details line (`layerSmartObjectBadgeButton`, embedded vs linked icon variants + `smartObjectLinked` property) is THE click affordance for opening the contents; the sibling `layerFxBadgeButton` opens the layer styles dialog, and row double-click opens layer styles for every layer kind, including smart objects (changed July 2026; the old behavior opened contents). The name and details labels yield horizontal space first and elide, so both fixed-size badge buttons stay visible at supported panel widths. Filter > Convert for Smart Filters converts the active normal pixel layer to an embedded Smart Object, after which Gaussian Blur, High Pass, Median, Dust & Scratches, or Surface Blur appends an editable native entry instead of changing its source pixels. All five direct filters remain destructive on ordinary pixel layers. Patchy's Tilt-Shift Blur is also destructive and gallery-capable, but has no native Smart Filter mapping yet. Smart Filter layer rows show the shared mask, stack and entry visibility controls, settings, Edit, Edit Blending Options, Duplicate, Move Up/Down, and Delete controls; execution order is stored bottom-up and displayed in reverse like Photoshop. Click the shared-mask thumbnail to paint it, Ctrl-click to load it as a selection, Shift-click to enable or disable it, and Alt-click for the grayscale mask-only view. The mask overlay and mask-only commands follow the Smart Filter mask while it is the active edit target. Unsupported imported stacks show disabled preservation controls. "Convert to Normal Layer (Rasterize)" (`layerSmartObjectToNormalAction`, in both Smart Objects menus) is a discoverable alias that just calls the Rasterize path. Painting the Smart Object's source pixels is refused with a status hint; Delete removes the object (like text); merge targets and rasterize/rasterize-style strip the smart-object data (`strip_layer_smart_object_data`); Export Smart Object Contents... (layer context menu) writes the embedded bytes verbatim. Duplicate, New Smart Object via Copy, and cross-document paste give each cloned layer a fresh `placed` uuid and clone/adopt its FEid/FXid record without copying the large backing bytes. A clone's FEid record must sit immediately after its source record, and its preserved Photoshop `lyid` block must be rekeyed to a fresh document-wide value; Photoshop 27.8 rejects an appended cache record and aliases layers that share `lyid`. Rasterize/delete remove the instance cache once no layer references it; undo restores the layer and cache together. A missing or ambiguous record blocks the clone instead of producing a Photoshop-corrupt layer.
- **Current Smart Filter editor limits**: Patchy authors and edits ordered Gaussian Blur, High Pass, Median, Dust & Scratches, and Surface Blur stacks. A new stack uses the exact current selection alpha as Photoshop's shared filter mask, or an explicit all-white mask when there is no selection; later entries retain that shared mask. Editable masks are full-document 8-bit grayscale targets. Brush, Eraser, Fill/Clear, Gradient, Line, Rectangle/Ellipse, and Invert edit the temporary mask buffer; unsupported tools are blocked rather than touching the Smart Object pixels. The model and native cache remain unchanged during a gesture, and each completed gesture commits one undo step and regenerates the filtered preview once. Each filter's More menu edits whole-percent opacity (Photoshop normalizes fractional values) and blend mode with cancellable live preview; Apply creates one undo step. Editable native masks are limited to documents of 64 megapixels or less; larger imported records remain preserved and preview-locked. A true `filterMaskLinked` flag is also preserved but preview-locked because Photoshop 27.8 did not produce a true-state fixture, so linked-mask editing is not exposed. Density and Feather are not exposed yet. Gaussian Blur and High Pass radii are fractional from 0.1 to 1000 px. New Gaussian Blur edits use the same practical 0.1 through 12 px range as the destructive dialog; an imported native value above 12 remains visible and is preserved when accepted unchanged. High Pass keeps its practical 0.1 through 12 px slider. Median stores fractional values from 1 to 500 px, renders their floored integer radius, and uses a practical 1 to 25 px slider. Dust & Scratches uses integer Radius 1 through 100, practical slider 1 through 25, and integer Threshold 0 through 255. Surface Blur stores a fractional radius from 1 to 100 px, rounds it to the nearest effective integer radius with a minimum of 1, uses a practical 1 to 25 px slider, and accepts integer Threshold 2 through 255. Larger native values remain preserved as described above. Preview/Cancel restores the exact cached raster, and each Apply, visibility, mask toggle, duplicate, reorder, or delete action produces one undo entry. The Visual Filters Gallery applies recipes containing only Gaussian Blur, High Pass, Median, Dust & Scratches, and Surface Blur non-destructively to a Smart Object; a recipe containing Tilt-Shift or any other Patchy-only filter asks for rasterization and is never partially mapped. Wrapping a filtered Smart Object inside another Smart Object is blocked until Patchy can transfer its native cache and document-space mask into the child file. Unsupported imported stacks remain byte-preserved and preview-locked.
- **Native-integrity safety gates**: Move-tool drags, keyboard nudges, Free Transform, and Warp rebuild a supported filtered Smart Object from its immutable source and refresh FEid. Those commits are transactional: a render/cache failure restores placement, pixels, native cache, and history. Whole-document Image Size, Canvas Size, Crop, and Rotate currently refuse documents containing any Smart Object because the native placement and document-space mask transforms are not calibrated yet. Layer Flip and Fill similarly refuse Smart Object pixels; use Free Transform or rasterize first. The Visual Filters Gallery maps fully supported recipes to native entries and offers to rasterize before applying any mixed recipe. Other destructive Image Adjustments, legacy plug-ins, Cut, Layer via Cut, Apply Layer Mask, Stroke Selection, indexed conversion, palette remapping, and palette snapping require rasterization before they can rewrite Smart Object preview pixels. Copy, Layer via Copy, ordinary mask editing, and metadata-only palette changes remain non-destructive and available. These gates also protect unsupported imported stacks whose Photoshop preview must remain byte-preserved.
- Opening a PSD with smart objects emits **import notices** (editable / preview-locked / external / dangling counts). These follow the status-bar-by-default rule (see docs/file-formats.md); UI tests opening such files through the window path only need the repeating dismisser when they enable `imports/showPsdWarningsAndInfo`. Tests reading via `DocumentIo::read_file` directly are unaffected (no dialogs).
- **Edit Contents**: clicking an editable embedded smart object's badge button (or Edit Smart Object Contents in the Smart Objects menus) opens the source as a child tab titled "file.ext (embedded in Parent.psd)". The child is a full `DocumentSession` whose `smart_object_link{parent_session_id, source_uuid}` carries session IDS (never pointers: sessions_ erases on close; `session_id` is the stable identity). Save (Ctrl+S) on the child routes `save_document()` to `commit_smart_object_child_session`: the child serializes in the source's own format (PSB for '8BPB' via `WriteOptions::large_document`, PSD for '8BPS', Qt image formats re-encode a flatten), the element gets fresh shared_ptr bytes (dirty), EVERY layer sharing the uuid re-renders (`smart_object_render.{hpp,cpp}`: quadToQuad + resample_transformed_rgba8, raster_status=patchy_raster), and the parent takes ONE undo snapshot ("Edit Smart Object Contents"); the child marks clean. Size-changing commits rescale quads per the E5 rule below; same-size commits leave SoLd untouched (only the lnk element regenerates). Closing a parent with open child tabs prompts (closeSmartObjectChildrenMessageBox) and resolves the children first; a child whose parent is gone detaches and falls back to Save As. Nested smart objects work by construction (each child is a full session; PSD children keep their own store through DocumentIo). Decode fidelity: PSD/PSB sources render their preview from the child's own flattened composite (prefer_flat_composite), so untouched children look exactly like Photoshop's stored pixels.
- **Preview decode falls back to the layered render when the stored composite is fully opaque but the layers are not** (`decode_smart_object_source_image`): a 3-channel or opaque composite cannot represent canvas transparency. Pre-fix Patchy children matted the composite onto black (the "transparent parts turned black on commit" bug), and Photoshop saves with Maximize Compatibility off write flat white-matted composites — both now render through the layered fallback, while genuinely opaque children keep the PS-exact stored composite. Regression pins: `ui_smart_object_edit_commit_keeps_canvas_transparency` (end-to-end convert/edit/commit), `ui_smart_object_legacy_black_composite_decodes_transparent` (committed pre-fix fixture `test-fixtures/psd/patchy-legacy-black-composite.psb`), and `ui_smart_object_psbtest_repro_decodes_transparent_if_available` (the original local repro file in `local-test-fixtures/psd/PSBtest/`). Parent files whose previews were already baked black heal on the next commit/transform/update re-render.
- **Edit-format guard**: PSD/PSB (DocumentIo both ways) and png/jpg/jpeg/tif/tiff/bmp/webp (Qt both ways) are editable; GIF and registry-only formats (TGA/PCX/...) decode for preview but refuse Edit/Replace with an explanatory status message (`classify_smart_object_contents`); undecodable/external sources keep Photoshop's preview forever. Content density comes from `smart_object_source_dpi` (PSD resolution or the image's embedded density, 72 fallback) — note Qt-authored pngs carry a pHYs chunk, so replacing with one honors its dpi like Photoshop does.
- **Replace Contents** follows the E5 semantics exactly: fresh uuid + element, every referencing layer repoints and rebuilds about its own quad center, old element removed, layer names swap the source stem. Accepted replace formats: psd/psb/png/jpg/jpeg/tif/tiff/bmp (webp only round-trips existing sources, its PS filetype OSType is unpinned).
- **Linked-file workflows**: external ('liFE') smart objects are fully workable. Edit Contents resolves the path (relPath against the parent folder, then a bare-filename sibling, then originalPath, then the fullPath URI: `resolve_smart_object_external_path`) and opens the REAL file as a disk-backed session with `SmartObjectLink::external`; its Save writes the file FIRST, then `refresh_external_smart_object_after_save` re-reads the bytes, stamps the liFE date/size, marks the element dirty, and re-renders every uuid-sharing layer as ONE parent undo step. Update Smart Object Content re-reads on demand; document open compares stored date/size and prepends "changed on disk"/"not found" notices (actionable notices go FIRST so the status bar shows them). Relink to File... follows PS's E14-captured semantics: like Replace Contents but staying external (a FRESH element uuid in the lnkE block, referencing layers repoint and rescale about their own centers per E5, layer names swap the source stem, old element pruned). Embed Linked follows E13: ANOTHER fresh uuid, the element moves to lnk2 as liFD, the emptied lnkE stays behind, locks clear, and the per-layer SoLE block key flips to SoLd (block_dirty repoints the Idnt on save). COM note: linked placement needs `putBoolean(charIDToTypeID("Lnkd"), true)` on the 'Plc ' descriptor. No QFileSystemWatcher by design (open-time check + explicit Update + self-save refresh cover the workflows). The shared re-render walk is `refresh_smart_object_layers_for_source`.
- **E4 acceptance**: Photoshop 2026 opened Patchy's committed, replaced, and nested-edit outputs (the `ui_smart_object_*.psd` test artifacts), color-sampled the re-rendered previews bit-exactly, opened each embedded contents, and resaved cleanly (no "disk error (-1)"). Re-run the e4_acceptance COM script over those artifacts after any writer-side smart-object change.
- **Authoring**: Convert to Smart Object (layer menu + context submenu) moves the selected layer trees into a child document (canvas = union of render bounds, PSB, '8BPB', named "<top layer>.psb"; nested sources adopt into the child store), the topmost selected layer's slot/id/name become the new smart object, and the preview composites pixel-identically for Normal/100% layers. Place Embedded (File menu) uses the E2 rule dpi-aware (physical size 1:1 centered, fit-to-canvas when larger). Both write a from-scratch SoLd via `psd::author_placed_layer_sold_payload`, which mirrors PS 2026's exact key order/id forms (E1 captures; `smart_object_authored_sold_matches_photoshop_shape` pins the shape against the committed fixture) and is pushed into the layer's unknown blocks at author time, so the normal preserve/patch-in-place machinery owns it from then on. Patchy writes NO legacy PlLd companion (PS writes both; it accepts SoLd-only files, E4-gated). New Smart Object via Copy clones the element under a fresh uuid (E8 semantics); free transform composes the delta into the quad and re-renders through `smart_object_transform_render_callback_` (text-layer pattern), with the resampled pixels as fallback.
- **Locked-but-parsed** external, legacy, and unsupported-warp layers still translate their quads on Move and regenerate patch-in-place; nonAffineTransform moves by the per-corner DELTA (never overwritten with the affine quad, which would flatten a perspective placement), and warpNone warp bounds stay the CONTENT rect (0,0,h,w) like PS writes them. An unsupported Smart Filter stack is pinned because its baked, document-space masked preview and native FEid cache cannot be moved independently. Truly unparseable SoLds import with lock="unparsed" and an EMPTY uuid (badge + paint/transform/move protection; `layer_has_movable_pixels` pins the layer because no quad can ride a move); the dangling-uuid cleanup skips that state and blocks re-emit verbatim forever.
- **Layer context menu**: reorganized into submenus (Layer Style / New / Smart Objects / Layer Mask + the Lock submenu) because the flat menu outgrew the screen. Seth's standing rule: **Edit Layer Styles... stays the FIRST item, always** (`ui_layer_context_menu_keeps_edit_styles_on_top` pins it). Test helpers search submenus recursively (`find_menu_action_by_text`).
- **Photoshop 2026 ground truth** (COM captures): Convert-to-SO writes BOTH PlLd and SoLd; child doc = "<layer name>.psb" ('8BPB', creator '8BIM'), canvas = layer pixel bounds, Trnf = those corners; Place uses filetype 'png ' etc. with creator four spaces, smaller-than-canvas places 1:1 centered, larger fits-to-canvas centered (`generalPreferences.placeRasterSmartObject` gates SO creation — it is FALSE on this machine, flip it temporarily in scripts and restore); lnk2 elements are version 8 with ~117 undocumented trailing bytes (parse by element length; Patchy writes v7); duplicates share the Idnt uuid, "New Smart Object via Copy" clones the element under a fresh uuid ('placed' is a per-layer instance uuid, always fresh); rasterizing in PS KEEPS the orphaned lnk2 element, so Patchy never prunes unreferenced sources either.
- **Smart Filter ground truth** (Photoshop 27.8 COM captures): `filterFXList` stores filters in application/execution order while the Layers panel displays them reversed; Gaussian Blur uses filter id `0x47736E42`, High Pass uses class/id `HghP` (`0x48676850`), Median uses class/id `Mdn ` (`0x4D646E20`), Dust & Scratches uses class/id `DstS` (`0x44737453`), and Surface Blur uses the full stringID class `surfaceBlur` with integer filter id 854 (`0x356`). Gaussian Blur, High Pass, Median, and Surface Blur preserve fractional `Rds ` pixel-unit values. Dust & Scratches instead stores integer `Rds ` followed by integer `Thsh`, and Photoshop's native entry name is literally `Dust && Scratches...`. Surface Blur stores its unit-double `Rds ` first and integer `Thsh` second. Photoshop native Tilt-Shift is not a standalone filter class: it is a `blurbTransform` Blur Gallery entry named `Blur Gallery...` with filter id 712 and a widget list that can contain several Blur Gallery tools. Patchy preserves that descriptor and FEid record but does not partially interpret or regenerate it. Per-entry blending stores a full-stringID mode plus percentage opacity. One-pixel line captures from Gaussian radius 0.1 through 8 pin Photoshop's small-radius kernels, FEid cache-canvas edge repeat, separability, premultiplied-alpha filtering, expanded visible bounds, and byte rounding after each pass. High Pass captures at 0.1, 1, 4.25, and 10 through 12 px pin straight RGB, unchanged alpha and bounds, repeated edges, hidden RGB behavior, and the `source - Gaussian + 128` formula. Median captures pin square edge-clamped channel medians, floored fractional radii, unchanged bounds, independent alpha filtering, and nearest-visible straight-RGB extension under transparent pixels. Dust & Scratches radius 7, threshold 23 pins its two-integer descriptor shape and Photoshop acceptance. Surface Blur captures pin nearest-integer effective radius, square edge-clamped sampling, per-channel weights `max(0, 5 * threshold - 2 * abs(v - c))`, ties-to-even quotient rounding, straight-RGB extension, the same formula on alpha, and at most radius-sized bounds growth. The canonical Surface Blur fixture pins Photoshop acceptance of the full-stringID descriptor. The two mixed Gaussian/Median fixture stacks are fully supported and pin native execution order; a separately corrupted native class keeps fail-closed unsupported-stack preservation covered. Each duplicate shares `Idnt` but has a distinct `placed` uuid and FEid record. Saving Edit Contents regenerates the shared `Idnt` and every referencing layer's `placed` uuid; Replace Contents does the same during the replace action. Rasterizing one duplicate leaves the surviving instance IDs unchanged. Photoshop 27.8 ignored attempts to set `filterMaskLinked=true`, so a true flag is preserved but keeps the imported layer preview-locked.
- **liFE (linked external file) element layout** (pinned from the 10cm-table-tent PS 2023 capture, v7 elements): uuid pascal string, unicode filename, filetype OSType, creator = four NUL bytes (not "8BIM"/spaces), u64 datasize (0), open descriptor flag + `{null; compInfo{compID:-1, originalCompID:-1}}`, then a SECOND u32 descVersion 16 + `ExternalFileLink` descriptor (class strID; keys in order: descVersion(str) long = 2, 'Nm  '(char) TEXT filename, fullPath(str) TEXT file:// URI, originalPath(str) TEXT native path, relPath(str) TEXT), then a 16-byte date struct {u32 year, u8 month, u8 day, u8 hour, u8 minute, f64 seconds} (the mod time PS compares for staleness), u64 file size, then the versioned tail every element kind shares: v5+ unicode child doc id, v6+ f64 assetModTime, v7+ u8 lock state. `parse_link_element` models all of it (best-effort try/catch: unmodeled variants degrade to the verbatim skip) and `serialize_external_element` mirrors it for dirty external sources; clean elements still re-emit byte-identically. `psb_life_trailer_fields_parse_if_available` + `smart_object_external_element_round_trips_if_available` pin both directions.
- **Replace Contents ground truth** (E5 COM captures, `ps2026_e5_*` local fixtures): PS creates a NEW element with a fresh uuid and repoints EVERY layer that referenced the old uuid (the replaced-away element is removed, unlike rasterize orphans); each layer's quad is rebuilt about its own quad center preserving the content-inch-to-doc-pixel linear map (`L_new = L_old * dpi_old/dpi_new`; same-dpi replaces degrade to pure pixel scaling), Sz/Rslt become the new content's pixel size and dpi, and layer names swap the old source stem for the new one ("A copy" becomes "B copy"). PS additionally rounds the resulting quad corners to whole pixels; Patchy keeps full doubles (a sub-pixel placement difference on OUR edit, not a render divergence). Place also writes an empty 'lnkE' block alongside 'lnk2'.

## Patents and trademarks (July 2026 assessment)

Assessed 2026-07-14: the smart object / smart filter architecture, the Gaussian Blur, High Pass, Median, and Dust & Scratches implementations, format/copyright law, and trademarks, then extended the same day for the newly landed Surface Blur and Tilt-Shift Blur. The binding summary lives in AGENTS.md "Legal constraints". This is engineering research, not legal advice; if any rights holder ever sends an actual letter, involve an IP attorney.

- **Architecture cleared**: no in-force Adobe patent was found covering embedded source documents rendered into a layer, re-editable parameterized filter stacks stored with a layer, per-filter blend mode/opacity, the shared filter mask, or a gallery-style preview UI. Claim-level sweep over Google Patents and FreePatentsOnline full text (assignees Adobe and Macromedia): "smart filter" appears in zero Adobe patent specifications, "smart object" in zero Adobe claims, "filter gallery" nowhere; the CS2 (2005, Smart Objects) and CS3 (2007, Smart Filters) filing windows are expired or in their final term-adjustment months, and sweeping those windows turned up nothing on point. The Gaussian Blur, High Pass, Median, and Dust & Scratches implementations are classic pre-1999 published techniques with no in-force coverage found.
- **Expired ancestors that now serve as prior art**: Adobe's adjustment-layer patents US 5974198 and US 6185342 (priority 1996-08-26, expired 2016), Live Picture/MGI's non-destructive editing US 6181836 (expired 2017), Microsoft's effect-graph layers US 7295208 (fee-lapsed, term ended 2026-05-26), and healing brush US 6587592 (Expired - Lifetime, 2021-11-19, so a classic texture/gradient healing brush is now unencumbered).
- **Surface Blur required a design-around; implemented 2026-07-14 with bit-identical output.** Adobe US 8594445 ("Fast bilateral filtering using rectangular regions", filed 2005-11-29, Active, adjusted expiration 2032-05-30 per Google Patents, 11.5-year maintenance fee paid May 2025) claims a bilateral filter whose spatial component is a 2-D box, computed by constructing a histogram of the window's pixel values and applying a range filter to that histogram centered at the center pixel's current value; the original implementation (sliding coarse/fine window histograms plus the triangle range weight) practiced every element of claim 1 and the sliding reuse of dependent claim 2, and it is almost certainly the patent covering Photoshop's own Surface Blur. Adobe US 7920741 ("integral histogram convolution", filed 2007-05-31, Active, adjusted expiration 2030-02-02) additionally claims the merge-column-histograms-then-slide-along-the-scanline structure itself, which the shared Median/Dust & Scratches machinery also practiced. The shipped design-around removes every histogram from Surface Blur (direct per-pixel accumulation of the same integer triangle-weighted sums up to `kSurfaceBlurDirectMaximumRadius` = 8 px; per-intensity-level box sums above it, the linear-shift-invariant decomposition published by Durand & Dorsey 2002) and rebuilt Median/Dust & Scratches on ONE window histogram updated a single pixel value at a time along a serpentine traversal (Huang 1979 prior art; no column histograms exist anywhere). Outputs stayed byte-identical: the calibrated `smart_filter_*` pixel tests passed unchanged and `surface_blur_direct_and_level_paths_match_reference` pins both new paths against the brute-force reference across the radius cutoff. NEVER reintroduce merged or slid window histograms in these filters, and never compute Surface Blur through ANY value histogram, before the respective expirations (2030-02-02 and ~2032-05-30; the US 8594445 figure could shorten to 2029-02-01 if a terminal disclaimer to its CIP child US 7783121 applies). Also examined and judged not to read: US 7783121 (value-range-limited histograms for HDR), US 7889923 (sparse merge skipping), US 8315473 (fractional-radius border regions), US 8655097 (brush-local bilateral with automatic tolerance), US 7675524 (enclosed-block associative convolution), US 10902569 (unique-value list pruning), US 7983511 (noise-space detail classification).
- **Tilt-Shift Blur's algorithm is clear of Adobe patents**: every independent claim in the CS6 Blur Gallery family requires an element Patchy lacks. US 8831371 needs two or more blur-pattern instances iteratively combined in one mask; US 9019310 needs field-blur pins feeding a per-pixel blur-RADIUS mask with per-pixel variable kernels; US 8693776 needs a user-variable selection-edge bleed; US 8824793 needs bokeh boost thresholds; US 8891942 is video moving-object tracking; the depth-map, motion-path, and deblurring families are facially inapplicable. Patchy's single linear pattern, 0..1 mix mask, and fixed tent-blur level blending sit outside all of them; Photoshop's on-canvas widgets appear only in specifications and dependent claims. No Adobe design patent on the Blur Gallery controls was found (all 88 Adobe "display screen" design patents reviewed). Imported Photoshop Blur Gallery smart filters stay preservation-only and preview-locked in Patchy, which creates no implementation exposure.
- **Tilt-Shift preview overlay was redesigned 2026-07-14 around a non-Adobe patent**: Apple US 8971623 ("Overlaid user interface tools for applying effects to image", priority 2012-03-06, Active, expires 2032-12-26) claims receiving a selected image location, overlaying a UI item comprising a first and second line flanking that location, each dividing the image into an area containing the location and one not, and applying the effect to the areas not containing it; dependent claims add parallel lines and name tilt-shift. The original overlay (two image-spanning solid focus lines plus two dashed full-blur lines around the center handle) plausibly practiced that. The shipped overlay marks each boundary with a short draggable grip bar instead (`kBoundaryGripHalfLength`, 26 px half-length, centered on the axis through the center handle); nothing spans or divides the image, while the center handle, rotation handle, hit-testing, and symmetric width editing are unchanged. `ui_filter_gallery_tilt_shift_overlay_uses_grip_bars` pins the no-spanning-lines rendering; never reintroduce image-spanning boundary lines before 2033. Strong invalidity context exists (Instagram 2011, Snapseed 2011, TiltShift Generator 2009, tiltshiftmaker ~2008 all showed on-image band UIs before the priority date) but stays fallback only. A systematic non-Adobe UI clearance was not performed; this patent surfaced incidentally, so give any new on-canvas gizmo its own check rather than assuming Adobe-only sweeps suffice.
- **Still-active gates for future features**: the PatchMatch / Content-Aware Fill trio US 8285055 (Active, adjusted expiration 2031-04-09), US 8340463 (Active, 2031-10-24), US 8355592 (Active, 2031-08-12). No content-aware fill, patch-based healing, or reshuffle-style synthesis before clearance or expiry. Active Adobe patents examined and judged not to read on the current smart filter feature set, with the adjacent feature that would trigger a re-read: US 8924851 (web-dev "web image objects" re-executing accumulated instructions, to 2032; re-read before any web-export non-destructive asset feature), US 8655112 (one-click merge-visible-layers + effect files + paintable mask on the merged layer, to 2032; Patchy's smart filters attach to ONE existing layer and must never auto-merge), US 9058699 (gradient-domain compositing solver, to 2029), US 8170380 (import/export keep-appearance vs keep-editable dual-state UI), US 7973950 (print-history reapply), US 11070717 (live-camera filter recommendation), US 11393077 + US 11763430 (GAN-trained dust/scratch removal, to ~2040; the classic median+threshold filter is outside the claims), US 11335049 (cloud component sync), US 12530821 (neural compositing in non-destructive workflows, granted Jan 2026; re-read before integrating generative output into smart-filter stacks).
- **Watch items**: pending Adobe application US 2022/0207115 A1 ("Determining and selecting operation features for digital content editing"); re-check its claims if it grants. Third-party histogram-acceleration patents also exist (e.g. US 8260076 "Constant time filtering"), so any future histogram-accelerated filter needs its own check, not just an Adobe-assignee sweep. Keyword clearance cannot categorically prove a negative, so keep doing per-feature checks like Quick Select's (docs/selection-tools.md); Adobe has asserted UI patents against a direct competitor before (Adobe v. Macromedia, 2000-2002, settled).
- **Format and copyright**: Adobe's "Photoshop File Formats Specification" (https://www.adobe.com/devnet-apps/photoshop/fileformatashtml/, November 2019 edition, confirmed live July 2026) is a public page with no click-through whose preface states it is provided for third parties to read and write the format; the restrictive 2004-2006 SDK EULA bound only its signatories. File formats are not copyrightable expression (Feist; 17 USC 102(b); Lotus v. Borland; Google v. Oracle; CJEU SAS Institute v. World Programming C-406/10), and Patchy's COM byte-diff method observes a licensed Photoshop's output without decompiling, so the reverse-engineering safe harbors (Sega v. Accolade, Sony v. Connectix, DMCA 1201(f), EU 2009/24/EC art. 5(3)/6) are belt-and-suspenders on top. Rules that keep this clean: no verbatim Adobe spec text in the repo (link it instead), self-authored fixtures only, no PSDC cloud-document support. Enforcement record: no documented Adobe action against any third-party PSD implementation or Photoshop clone (GIMP, Krita, Affinity, Corel, Apple, Photopea all ship PSD support; Photopea has sold a direct in-browser clone since 2013 untouched); Adobe's actual enforcement targets are piracy, DRM circumvention, cybersquatting, and trademark misuse.
- **Trademarks**: "Smart Object(s)", "Smart Filter(s)", "Filter Gallery", "Healing Brush", "Content-Aware", and "PSD" are absent from Adobe's published trademark list (May 21, 2026 edition; also absent from the 2009 edition) and from all 601 Adobe-owned US trademark records; the existing SMART FILTER / SMART OBJECT registrations belong to unrelated companies. PHOTOSHOP is registered (e.g. Reg. 1,850,242, renewed) and policed against generic-verb use and cybersquatting. Photopea uses the feature names verbatim commercially; Krita/Affinity rename theirs (File/Filter Layers, Live Filters), the conservative norm. Patchy's usage stays nominative (the terms describe Photoshop's own stored data), product branding contains no Adobe marks, and README.md carries the attribution notice in Adobe's preferred format. UI look-and-feel claims are weak doctrine (Apple v. Microsoft, 35 F.3d 1435 (9th Cir. 1994); Lotus v. Borland, 49 F.3d 807 (1st Cir. 1995)) and no Adobe UI trade-dress enforcement was found; the gallery's own name ("Visual Filters & Looks") and original tool icons keep extra distance.
